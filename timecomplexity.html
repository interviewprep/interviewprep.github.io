Time Complexity

Why is time complexity important?

Many internet companies deal with large quantities of data. The difference between a O(n^2) algorithm and O(n) algorithm can be significant when dealing with massive data. 

Of course, in programming interviews, coming up with a solution with a lower time complexity is also seen as demonstration of problem solving skills. 

The intuition behind time complexity:

 

Time complexity is not about how much time a program takes to execute, rather how the execution time scales with input.

Guess the time complexity?

What of the following time complexities best applies to the following functions?

O(n), O(log n), O(n log n), O(n^2), O(n!), O(n log log n)

 

Easy questions:

1.

int  fun(int n) {

int count = 0;

for (int i=0;i<n; i++) {

  for (int j=0;j<n; j++) {

    count = count + 1

}

}

for (int i =0; i<n; i++) {

  count = count + 1;

}

return count;

}

 

2.

int fun(int n)

{

int count = 0;

 for (int i = n; i > 0; i=i/2) {

       count = count + 1;

 }

 return count;

}

 

Challenging questions:

 

3.

int fun(int n)

{

 int count = 0;

 for (int i = n; i > 0; i /= 2) {

    for (int j = 0; j < i; j++) {

       count += 1;

    }

}

 return count;

}

 

4.

void fun(int n, int arr[])

{

   int i = 0, j = 0;

   for(; i < n; ++i) {

       while(j < n && arr[i] < arr[j]) {

           j++;

        }

    }

}

 

 

5.

int fun(int n)

{

int count = 0;

 for (int i = 0; i < n; i++) {

    for (int j = i; j > 0; j--)   {

       count = count + 1;

     }

   }

 return count;


}

 

 

Answers

 

  1. O(n^2)
  2. O(log n)
  3. O(n)
  4. O(n)
  5. O(n^2)
